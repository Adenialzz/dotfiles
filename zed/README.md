**SendKeystrokes 要拆成空格分隔**

因为 Zed 这里不是在解析 Vim 命令字符串，而是在解析一串按键事件。

官方文档对 keymap 的定义就是：按键序列用空格分隔；SendKeystrokes 发送的也是一串按键，所以要写成一个个 keypress。也就是说，"1 0 j" 表示“依次按下 1、0、j”，而不是一整个文本 "10j"。

再直白一点：

"ctrl-k ctrl-s" 在 Zed 里表示先按 Ctrl+K，再按 Ctrl+S
同理，"1 0 j" 表示先输入 1，再输入 0，再输入 j

所以：

"10j"：更像一个单独 token，Zed 不会自动替你拆成 1 0 j
"1 0 j"：明确告诉 Zed 这是三个连续按键，于是 Vim normal mode 收到的就是 10j 这个操作

本质上是这区别：

你以为：传的是“命令字符串”
Zed 实际上：传的是“按键流”

所以它要求拆开写。
