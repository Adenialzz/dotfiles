
# Coding Agent 须知

为了更好地与用户协作、提高工作效率、对齐用户偏好，你必须牢记以下要求和建议。

## 行为要求

- 你不能总是假设用户非常清楚自己想要什么和该怎么得到。请保持审慎，从原始需求和问题出发，如果动机和目标不清晰，停下来和用户讨论。如果目标清晰但是路径不是最短，告诉用户，并且建议更好的办法

- 不要直接写代码！每次用户提出改动要求后，先和用户讨论确定技术选型和实现方案

- 每次大的新增或改动功能完成后，如果你认为有必要的话，主动修改 README.md 和 AGENTS.md / CLAUDE.md.


## 代码风格

**简洁性**

- 写代码时，在方案合理的前提下，用尽量少的改动完成功能

- 能不用配置就不用配置，能不用抽象就不用抽象

- 坚持 fail fast 原则，不要用大量 try except，让问题难以溯源

- 代码实现中不允许给出任何过度设计的兼容性或补丁性的代码，保持核心实现简洁。比如下面这种，就是不需要的：

    ```python
    if concurrency <= 0:
        raise ValueError("--concurrency 必须 >= 1")
    ```


**语言偏好**

- 除非用户特别指定，一律用中文进行对话和写文档


## 安全

- 为了避免在用户不知情的情况下丢失未提交的代码改动，请你每次开始写代码之前，先确认 git 工作区是否存在已有改动。如果存在已有改动，在开始编写代码前需要先提醒用户并向用户确认。

    - 本轮中你自己的改动，可以忽略

    - 本次用户没有要求写代码，可以忽略

- 如果涉及大批量数据处理或高并发请求，不要自己直接运行。将脚本写好后先通知用户

## 网络环境

- 如果遇到网络环境不通或者超时等问题，请检查 `http_proxy`、`https_proxy`、`no_proxy` 这几个环境变量。

## 特定编程语言注意事项

### Python

- 用 uv （而非原生的 python 命令）来进行 python 包管理和运行 python 代码或测试
